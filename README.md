# 教科書シェア v12

学生同士で不要になった教科書を、金銭の仲介なしで譲渡する流れを想定した3層構造のWebアプリです。公開デモに掲載する利用者・教科書・教科・教員・チャットは、すべて架空データへ統一しています。

## 公開デモ上の注意

- 掲載情報はすべて授業課題用の架空データです。
- 実在する個人名、授業名、教員名、メールアドレスなどは登録しないでください。
- ログイン方式は授業用の簡易方式であり、本番サービス向けの認証ではありません。

## v12の主な改善

- 既存の公開DBを全削除して架空データだけを再投入できる安全確認付きコマンドを追加
- アカウント、教科書、著者、教科、教員、チャット、通知などを明確な架空名へ統一
- 全画面に公開デモであることを示す案内を追加
- ログイン画面に架空のデモアカウント入力ボタンを追加
- ナビゲーションやログイン済み画面でメールアドレスを不用意に表示しないよう修正
- マイページ上部を「新規更新・出品中・問い合わせ中・お気に入り」の操作付きカードへ再設計
- 件数を「○件」と大きく表示し、次に行う操作を明示
- 保存検索、最近見たもの、履修科目、未読通知を補助リンクとして整理
- PWAキャッシュをv12へ更新

## 架空デモデータ

`npm run db:reset-demo` により、現在のDB内データをすべて削除した後、次の架空データを投入します。

- 架空アカウント：24件
- 架空教科書：40冊
- 架空教科：約50件
- 出品：80件
- 架空チャット：約100件
- 履修登録：約120件
- お気に入り：約140件
- 閲覧履歴：約190件
- 保存検索：24件
- 通知：18件

名称は「デモ利用者A01」「蒼海情報構造演習」「デモ教員A01」など、実在情報と区別しやすい形式に統一しています。

## デモ用アカウント

パスワードはありません。ログイン画面のボタンから自動入力できます。

```text
出品者として試す
デモ利用者A02 / webpro-demo-a02@keio.jp

受取希望者として試す
デモ利用者A01 / webpro-demo-a01@keio.jp
```

## 主な機能

- 名前・大学メールアドレスによる簡易ログインとアカウント作成
- 教科書の出品、編集、取り消し、譲渡完了
- 既存教科書候補の選択と関連情報の自動入力
- 教科書名・教科名・著者名・担当教員名による検索
- 本の状態、取引状態、並び順による絞り込み
- 譲渡品ごとのチャットと交渉相手の選択・変更
- お気に入り、閲覧履歴、保存検索、通知
- 履修科目・お気に入り・閲覧履歴を使ったおすすめ
- PWA、スマートフォン表示、キーボード操作への対応

## 3層構造

- **フロントエンド**：EJS、HTML、CSS、JavaScript
- **バックエンド**：TypeScript、Express
- **データベース**：PostgreSQL、Prisma

## 主な画面

| URL | 内容 |
| --- | --- |
| `/` | 検索、パーソナライズされた棚、譲渡品一覧 |
| `/login` | ログイン、新規アカウント作成、デモアカウント入力 |
| `/items/new` | 教科書の出品 |
| `/items/:itemId` | 譲渡品詳細、チャット、交渉操作 |
| `/items/:itemId/edit` | 自分の出品内容の編集 |
| `/mypage` | 更新、出品、問い合わせ、お気に入り、履歴、保存検索、履修科目 |
| `/notifications` | アプリ内通知一覧 |

## 通常の起動

```bash
npm install
npm run db:generate
npm run db:push
npm test
npm start
```

## 公開DBを架空データへ完全置換

この操作は、`DATABASE_URL` が指すDBの既存データをすべて削除します。

```bash
CONFIRM_DEMO_RESET=YES npm run db:reset-demo
```

誤操作防止のため、`CONFIRM_DEMO_RESET=YES` がない場合は実行されません。Renderの公開DBを対象にするときは、`.env` の `DATABASE_URL` が対象DBを指していることを必ず確認してください。

## Render

例として次の設定を使用します。

```text
Build Command:
npm clean-install && npx prisma generate && npx prisma db push

Start Command:
npm start
```

DBの全削除・架空データ投入はBuild Commandへ入れず、必要なときだけ手動で実行してください。デプロイのたびにデータが初期化される事故を防ぐためです。
